What is Moduleco?

Moduleco is a modular multi-agent plateform designed to simulate markets, organizations, social phenomenon and population dynamics.

It is implemented in Java, and runs on all platforms with a Java Virtual Machine (Java version 1.4 or higher).

  • Development Status: 1.4.5
  • Environment: Java
  • Intended Audience: Economists
  • License: GNU General Public License (GPL)
  • Operating System: Windows, GNU/Linux, MacOS, etc.
  • Programming Language: Java

Download

You can download the last version of Moduleco at http://www.cs.man.ac.uk/ai/public/moduleco/index.php#download.

Install

Just unzip the zip file. A new directory, called moduleco, will be created. The structure of this directory should look like that:

moduleco
     libLibraries (.jar files) for the platform and some demonstration models
     modelsModels (.java source files)
     outputsOutput of the simulations (.csv Excel files)
     docsDocumentation
     srcPlatform source files
     buildExecutables (.class files)
    moduleco.batWindows launch script
    moduleco.shLinux launch script
    README.htmlReadMe file

Run

Launching the platform

To launch the platform Moduleco, use the following script, depending on your Operating System:

Windows:moduleco.bat
Linuxmoduleco.sh


The initial User Interface

Once Moduleco is launched, you are greeted with the basic interface. You can now:

  • Select a model (using the top-right list of Models)
  • Start the simulation (using the top-left button Start)
  • Stop the simulation (using the top-left button Stop)
  • Run the simulation step-by-step (using the top-left button MStep)
  • Reinitialise the simulation (using the top-left button Create)

Now feel free to familiarise yourself with the more advanced options provided by the platform.

Running scripts

You can also launch Moduleco in command line, without User Interface. This is extremely convenient to run batches of simulations by night, and explore the parameters space of your models. The syntaxe is as follows:

moduleco.bat --noGUI $MODEL

where $MODEL is the name of a model available.

The number of simulations to run, their parameters, etc. are defined in modulecoFramework.Core and $MODEL.Autorun . The results of the simulation are automatically recorded in the outputs directory.

Build your own models

Models are all stored in the models directory:

modelsDirectory containing all the models
     heatbugHeatbug model
     isingIsing model
     lifeLife model
     ...

1. Creating your model

To create your own model, say myModel:

  • Create the sub-directory myModel in models.
  • Add the source code of your own classes (classes World and Agent are mandatory)
  • Copy the file compile.bat (under Windows) or compile.sh (under Linux) to myModel
modelsDirectory containing all the models
     myModelDirectory containing my source files
         World.javaSource code for the class World
         Agent.javaSource code for the class Agent
         ...
         compile.batScript to compile your code

2. Compiling

To compile your source code (.java files), go in myModel and run the compile script:

cd models/myModel
compile.bat

The binary files (.class) corresponding to the source files (.java) should appear in the build directory:

build
     models
         myModel
             World.classBinary for the class World
             Agent.classBinary for the class Agent
             ...

3. Executing

To execute your code, simply run Moduleco.

cd ../..
moduleco.bat (or moduleco.sh under Linux)

Your model should appear in the list of available models.
